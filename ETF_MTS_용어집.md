# ETF MTS Demo - 용어집 & 기술 가이드



## 기본 Role 정의
1. 자산운용사 (ETF 발행사)
자산운용사는 ETF 상품의 기획, 운용, 관리를 책임집니다. 
상품 설계 및 상장: 기초지수를 선정하고 이를 추종하는 ETF 상품을 만들어 증권시장에 상장합니다.
자산 운용: 투자자들의 자금을 모아 지수 구성 종목들을 실제로 매수하고 관리합니다.
정보 공시: 매일 ETF의 자산구성내역(PDF)과 순자산가치(NAV)를 산출하여 투자자에게 알립니다.
수익 분배: 운용 보수를 수취하고, 배당금 등 발생 수익을 투자자에게 분배금으로 지급합니다. 

2. 증권사 (LP: 유동성 공급자, Liquidity Provider) 
증권사는 자산운용사와 계약을 맺고 ETF 거래가 시장에서 원활히 이루어지도록 유동성을 공급합니다. 
호가 조성: 매수하려는 사람과 매도하려는 사람이 없을 때, 적정 가격(NAV 부근)에 매수/매도 호가를 지속적으로 제시합니다.
가격 괴리율 관리: 시장 가격이 ETF의 실제 가치(NAV)보다 너무 높거나 낮아지지 않도록 조절하여 투자자가 불리한 가격에 거래하지 않게 돕습니다.
유동성 공급 의무: 한국거래소 규정에 따라 장중 일정 시간 동안 의무적으로 호가를 제출해야 합니다.

#### 요약 비교
구분 	역할	비유
자산운용사	ETF 제작 및 자산 운용	제품 생산 공장 (제조사)
증권사(LP)	매수/매도 호가 제시 및 거래 지원	마트/편의점 (유통사)

#### 괴리율 관리 측면
1. 증권사 (LP) - 실행 및 유동성 공급 책임
호가 조성 의무: LP는 장중에 실시간 순자산가치(iNAV)에 가까운 가격으로 매수·매도 호가를 제시하여 괴리율이 벌어지지 않도록 직접적인 역할을 수행합니다.
평가 및 제재: 괴리율이 일정 범위를 벗어나 관리 의무를 다하지 못할 경우, 한국거래소로부터 낮은 평가 등급을 받거나 LP 자격에 제한을 받는 등 실질적인 페널티를 입게 됩니다. 

2. 자산운용사 - 공시 및 모니터링 책임
괴리율 공시: 장 중 괴리율이 일정 수준(국내 투자 ETF 1%, 해외 투자 ETF 2%)을 초과하여 발생할 경우, 자산운용사는 이를 즉시 한국거래소 기업공시채널(KIND) 등을 통해 투자자에게 공시해야 할 의무가 있습니다.
LP 모니터링: 자산운용사는 계약을 맺은 LP가 호가를 적절히 공급하고 있는지 상시 모니터링하며, 문제가 생길 경우 LP에 시정을 요구해야 합니다. 

3. 한국거래소 - 시장 감시 및 규제 책임
투자유의종목 지정: 괴리율이 비정상적으로 확대된 상태(예: 국내물 6%, 해외물 12% 이상)가 지속되면 해당 ETF를 투자유의종목으로 지정하거나 거래를 일시 정지시켜 투자자를 보호합니다. 

요약: 누가 어떤 책임을 지나요? (2026년 기준)
주체 	책임 내용
증권사(LP)	실시간 호가 제시를 통해 괴리율을 직접 억제할 의무
자산운용사	괴리율 초과 시 공시 의무 및 전반적인 운용 관리 책임
한국거래소	시장 가격 왜곡 감시 및 투자유의종목 지정 등 시장 관리 책임




## Part 1: ETF 핵심 지표 용어

### 가격 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **현재가** | Price | 지금 거래되고 있는 가격 |
| **전일종가** | Previous Close | 어제 장 마감 시점 가격 |
| **등락률** | Change % | 전일 대비 몇 % 올랐는지/내렸는지 |
| **iNAV** | Indicative NAV | 실시간 추정 순자산가치. "이 ETF가 진짜 이 가격 가치가 있는지" 판단 기준 |

---

### 건전성 지표

| 용어 | 영문 | 설명 | 좋은 수준 |
|------|------|------|----------|
| **괴리율** | Discrepancy | 현재가와 iNAV의 차이(%). 클수록 "거품" 또는 "저평가" | ±1% 이내 |
| **스프레드** | Spread | 매수호가와 매도호가의 차이(%). 좁을수록 거래비용 적음 | 0.5% 이내 |
| **TER** | Total Expense Ratio | 총보수율. 1년간 운용비용 | 0.5% 이내 |
| **거래대금** | ADTV | 하루 평균 거래 금액. 높을수록 사고팔기 쉬움 | 10억 이상 |
| **건전성 점수** | Health Score | 위 지표들을 종합한 0~100점 점수 | 70점 이상 |

---

### 수익 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **배당수익률** | Dividend Yield | 1년간 받는 배당금 / 현재가 (%) |
| **추적오차** | Tracking Error | 목표 지수와 실제 수익률 차이. 낮을수록 정확히 따라감 |
| **변동성** | Volatility | 가격이 얼마나 출렁이는지. 높으면 위험도 높음 |

---

### 상품 분류

| 용어 | 설명 |
|------|------|
| **레버리지** | 지수 변동의 2배로 움직임 (상승 2배, 하락도 2배) |
| **인버스** | 지수와 반대로 움직임 (지수 하락 시 수익) |
| **환헤지** | 환율 변동 위험을 제거한 상품 (H 표시) |
| **국내/해외** | 투자 대상 시장 구분 |
| **자산군** | 주식 / 채권 / 원자재 / 통화 등 투자 대상 종류 |

---

### 계좌 종류

| 계좌 | 세율 | 특징 |
|------|------|------|
| **일반계좌** | 15.4% | 제한 없음, 레버리지/인버스 가능 |
| **ISA** | 9.9% | 비과세 한도 있음, 레버리지 제한적 |
| **연금계좌** | 5.5% | 세금 혜택 최대, 레버리지/인버스 불가 |

---

### 시장 참여자

| 용어 | 영문 | 역할 |
|------|------|------|
| **LP** | Liquidity Provider | 유동성 공급자. 호가를 제시하여 거래가 원활하게 이뤄지도록 함 |
| **AP** | Authorized Participant | 지정참가회사. ETF 설정/환매 담당 |
| **운용사** | Asset Manager | ETF를 만들고 운용하는 회사 (삼성, 미래에셋 등) |

## Part 2: 이 앱의 주요 기능

### 홈 화면
- **포트폴리오 요약**: 계좌별 총 평가금액, 수익률
- **인기 ETF**: 거래대금 기준 실시간 TOP 5
- **테마 히트맵**: 12개 테마별 주간 수익률 시각화
- **시장 현황**: KOSPI, S&P500 등 주요 지수

### 탐색 화면
- **검색/필터**: 종목명, 테마별 필터링
- **정렬**: 건전성순, 저비용순, 유동성순
- **3가지 모드**: 탐색(카드) / 검증(테이블) / 주문

### ETF 상세
- 가격 차트, 건전성 점수
- TER, 괴리율, 스프레드 상세
- 상품 개요, 운용 전략

### 비교 화면
- 최대 3개 ETF 동시 비교
- 주요 지표 나란히 비교

### 투자정보
- ETF 101 학습 콘텐츠
- 용어사전
- 리서치 PDF

## Part 3: 기술 스택 초보 가이드

### "이게 뭘로 만들어졌나요?"

```
┌─────────────────────────────────────────────────────┐
│                    사용자 브라우저                    │
│                  (크롬, 사파리 등)                    │
└─────────────────────────────────────────────────────┘
                          ▲
                          │ 인터넷
                          ▼
┌─────────────────────────────────────────────────────┐
│                   Vercel (호스팅)                    │
│            웹사이트를 인터넷에 올려주는 서비스          │
│                 etf-mts-demo.vercel.app             │
└─────────────────────────────────────────────────────┘
                          ▲
                          │ 배포 (업로드)
                          ▼
┌─────────────────────────────────────────────────────┐
│                    빌드된 파일                        │
│         HTML + CSS + JavaScript (압축됨)            │
└─────────────────────────────────────────────────────┘
                          ▲
                          │ 빌드 (변환)
                          ▼
┌─────────────────────────────────────────────────────┐
│                   개발 소스코드                       │
│              TypeScript + React + CSS               │
└─────────────────────────────────────────────────────┘
```

---

### 핵심 기술 용어

| 용어 | 비유 | 설명 |
|------|------|------|
| **TypeScript** | 한국어 맞춤법 검사기 | JavaScript에 타입 검사 추가. 오타/버그 방지 |
| **React** | 레고 블록 | 화면을 "컴포넌트" 단위로 조립. 재사용 가능 |
| **Vite** | 번역기 + 압축기 | 개발 코드를 브라우저가 읽을 수 있게 변환 |
| **TailwindCSS** | 인테리어 도구 세트 | 색상, 크기, 배치 등을 쉽게 지정 |
| **shadcn/ui** | 이케아 가구 | 미리 만들어진 버튼, 카드 등 UI 부품 |
| **Vercel** | 부동산 임대 | 웹사이트를 인터넷에 올려주는 호스팅 서비스 |

### 호스팅이란?

> **비유**: 식당을 차리려면 건물(서버)이 필요한데, 직접 사지 않고 빌리는 것

| 방식 | 설명 | 예시 |
|------|------|------|
| **서버리스 호스팅** | 서버 관리 없이 코드만 올리면 됨 | Vercel, Netlify |
| **클라우드 서버** | 가상 컴퓨터를 빌려서 직접 관리 | AWS EC2, GCP |
| **자체 서버** | 회사 내 물리적 서버 운영 | 사내 IDC |

이 프로젝트는 **Vercel (서버리스)** 사용 → 무료, 간편, 자동 배포

---

### 배포 과정

```bash
# 1. 코드 수정 후

# 2. 빌드 (개발코드 → 배포용 파일)
npm run build

# 3. Vercel에 업로드
npx vercel --prod

# 4. 끝! 자동으로 URL 생성됨
# https://etf-mts-demo.vercel.app
```

---

### 프론트엔드 vs 백엔드

| 구분 | 설명 | 이 프로젝트 |
|------|------|------------|
| **프론트엔드** | 사용자가 보는 화면 | React (TypeScript) |
| **백엔드** | 서버, 데이터베이스 | 없음 (목업 데이터) |

> 현재는 **프론트엔드만** 있는 데모입니다.
> 실제 서비스 시 백엔드 API 연동 필요.
